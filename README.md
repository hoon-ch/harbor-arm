# Harbor ARM64

Automated builds of [Harbor](https://github.com/goharbor/harbor) for ARM64 architecture.

[![Build Status](https://github.com/hoon-ch/harbor-arm/workflows/Check%20and%20Build%20Harbor%20ARM64/badge.svg)](https://github.com/hoon-ch/harbor-arm/actions)

## Overview

This repository is a downstream Harbor ARM64 build and distribution repo. It
tracks official Harbor image tags to find stable Harbor image versions, checks
out the matching `goharbor/harbor` release source, rebuilds selected Harbor
components on ARM64 runners, and publishes the resulting images to Docker Hub
and GitHub Container Registry.

Harbor upstream publishes multi-architecture images for recent releases. Use
the official `goharbor/*` images unless you specifically need this repository's
rebuilt images, naming convention, or release artifacts. This repo exists to
make the ARM64 rebuild pipeline explicit, inspectable, and reproducible.

## Features

- **Automated Release Detection**: Daily checks for new stable Harbor image tags
- **ARM64 Native Builds**: Builds run on native ARM64 GitHub-hosted runners
- **Component Matrix**: Built components are declared in `scripts/config.sh`
- **Multi-Registry Publishing**: Docker Hub and GitHub Container Registry
- **Validation Pipeline**: Image existence and architecture checks, optional standalone smoke checks, inline Trivy security scan summaries, and uploaded validation markdown reports

## When to Use This Repository

Use these images only when you want an inspectable downstream ARM64 rebuild or
the `harbor-*-arm64` naming convention. For most production deployments, first
evaluate the official `goharbor/*` multi-architecture images.

## Quick Start

### Pull Pre-built Images

```bash
# Latest version
docker pull hoon-ch/harbor-core-arm64:latest

# Specific version
docker pull hoon-ch/harbor-core-arm64:2.15.1

# Published components use the -arm64 suffix:
# prepare, core, db, jobservice, log, nginx, portal,
# redis, registry, registryctl, exporter
```

### Kubernetes Deployment

For Kubernetes deployment experiments:

```bash
# Deploy the example configuration
kubectl apply -f examples/kubernetes/harbor-production.yaml

# Get LoadBalancer IP
kubectl get svc harbor-nginx -n harbor
```

**See [Production Deployment Guide](examples/kubernetes/PRODUCTION_DEPLOYMENT.md) for complete instructions.**

### Docker Compose

The Docker Compose file in `examples/docker-compose/harbor-arm64.yml` is a
reference for Harbor configuration generated by `prepare`, not a standalone
one-file deployment. It expects generated `common/config` content plus the
required secret and data directories mounted in the paths used by the compose
file.

Use the official Harbor `prepare` workflow to generate the required
configuration and directories before adapting the compose example. The
repository deployment docs will be updated as part of the hardening work.

## How It Works

1. **Daily Checks**: GitHub Actions runs daily at 00:00 UTC
2. **Release Detection**: Checks official `goharbor/harbor-core` image tags for the highest stable Harbor version
3. **Automated Build**: Triggers ARM64-native build on ubuntu-24.04-arm runners
4. **Validation Pipeline**: Validates image existence and architecture, runs optional standalone smoke checks, includes inline Trivy security scan summaries, and uploads validation markdown reports
5. **Multi-Registry Push**: Publishes versioned images to Docker Hub and GHCR; updates `latest` only for the highest stable Harbor image version
6. **Version Tracking**: Updates `built_versions.txt` to prevent duplicate builds

See [Architecture Documentation](docs/architecture.md) for technical details.

## Manual Build Trigger

To build a specific version:

```bash
# Using GitHub CLI
gh workflow run build-harbor-arm64.yml -f version=v2.15.1

# Or via GitHub Actions UI:
# https://github.com/hoon-ch/harbor-arm/actions
```

## Requirements

- **Docker**: 20.10+ with ARM64 support
- **Kubernetes**: 1.24+ (for k8s deployment)
- **ARM64 Hardware**: Native ARM64 processor or cloud instance
- **Memory**: Minimum 4GB RAM recommended
- **Storage**: Varies by deployment (10GB+ recommended)

## Security

- All images are built from official Harbor source code
- Trivy security scanning in CI/CD pipeline
- Regular updates following Harbor releases
- Report security issues through [GitHub Issues](https://github.com/hoon-ch/harbor-arm/issues) unless a private disclosure path is added.
